# Direction-of-play correctness — Sportec + Metrica per-period-absolute fix — PR-S23 — silly-kicks 3.0.1

**Status:** Approved (design)
**Target release:** silly-kicks 3.0.1
**Author:** Karsten S. Nielsen with Claude Opus 4.7 (1M)
**Date:** 2026-05-02
**Predecessor:** silly-kicks 3.0.0 — PR-S22 (direction-of-play correctness refactor; ADR-006)
**Triggered by:** luxury-lakehouse SK3-MIG Group A migration session — empirical lakehouse probe against silly-kicks 3.0.0 surfaced that native `silly_kicks.spadl.sportec` + `silly_kicks.spadl.metrica` hard-code `declared=ABSOLUTE_FRAME_HOME_RIGHT` while real bronze events for both providers ship `PER_PERIOD_ABSOLUTE` (teams switch ends after halftime). PR-S22's contract-only test layer missed the bug; the validator's "alternates between periods" detector additionally false-positives on sparse-shot matches.

---

## 1. Problem

### 1.1 Bug (silly-kicks 3.0.0)

`silly_kicks/spadl/sportec.py:516-535` and `silly_kicks/spadl/metrica.py:169-194` both hard-code:

```python
validate_input_convention(events, declared=ABSOLUTE_FRAME_HOME_RIGHT, ...)
actions = to_spadl_ltr(actions, input_convention=ABSOLUTE_FRAME_HOME_RIGHT, home_team_id=...)
```

Empirical evidence from lakehouse production fixtures (Bassek et al. DFL DataHub `idsse_J03WMX`; Metrica Sample Game 1) shows both providers' raw events ship `PER_PERIOD_ABSOLUTE`: per-period-per-team mean shot start_x clusters near one goal in P1 and near the opposite goal in P2 — the textbook signature of teams switching ends.

Net effect of the wrong declaration: `_mirror_absolute_frame` flips away-team rows uniformly across periods. For per-period-absolute input, this leaves both teams clustered at the *wrong* end in alternating periods. SPADL canonical convention ("all teams attack left-to-right") fails on Sportec + Metrica output for half of every match.

The Metrica converter's validator did emit a `confidence=medium` warning ("per_period_absolute") on the lakehouse fixture but only at warn-level (lakehouse runtime did not have `SILLY_KICKS_ASSERT_INVARIANTS=1` set). The Sportec converter's validator did NOT warn on the same fixture — the IDSSE shot distribution (4/5/8/3 across team×period) is sparse enough that only 2 of 4 (team, period) groups meet the detector's `min_shots_per_group_medium=5` threshold, and each team has only one reliable group, which short-circuits the `per_team_sides` analysis to false-positive ABSOLUTE_FRAME_HOME_RIGHT.

### 1.2 Why PR-S22 missed it

ADR-006 § 6 formalised three test layers (contract / physical-invariant / end-to-end). The invariant-test layer was added but the existing fixtures used by `tests/invariants/` had insufficient per-period shot density to expose the per-period bug:

- `tests/datasets/idsse/sample_match.parquet`: 308 rows, 2 periods, **only 2 shots total** — too sparse for any per-period orientation assertion.
- `tests/datasets/metrica/sample_match.parquet`: 300 rows, **period 1 only** — physically incapable of catching a per-period-flip bug.

PR-S22's invariant tests therefore covered StatsBomb / Wyscout / kloppy gateway end-to-end but had blind spots on the two native converters that are now broken in 3.0.0.

### 1.3 Migration-readiness signal

silly-kicks's TRACKING-side Sportec adapter at `silly_kicks/tracking/sportec.py:53-128` already accepts `home_team_start_left: bool` as a required kwarg and uses `_direction.home_attacks_right_per_period(...)` to derive per-period flips — the exact PFF pattern, applied to Sportec, in production. The events-side converter just missed mirroring this contract during the PR-S22 refactor. Lakehouse-side bronze loaders that already extract `home_team_start_left` for tracking can mirror that derivation for events with zero new infrastructure.

## 2. Goals

1. **Native Sportec + Metrica converters declare `PER_PERIOD_ABSOLUTE`** and dispatch to `to_spadl_ltr(... PER_PERIOD_ABSOLUTE, home_attacks_right_per_period=flips)` correctly. SPADL canonical convention restored on both providers.
2. **API-additive** new keyword-only kwargs on `sportec.convert_to_actions` + `metrica.convert_to_actions`: `home_team_start_left`, `home_team_start_left_extratime`, `home_attacks_right_per_period`. Mutual exclusion + loud-failure default. Mirrors the existing PFF events-side and tracking-side Sportec API exactly.
3. **Real-fixture invariant test layer** (`tests/invariants/test_per_period_orientation.py`) parametrized over all five native converters + the kloppy gateway, with `xy_fidelity_version ∈ {1, 2, None}` axis on the StatsBomb path. Asserts both per-period AND aggregate canonical-LTR invariants.
4. **Lakehouse fixtures committed** under existing `tests/datasets/idsse/` + `tests/datasets/metrica/` directories with new `per_period_match.parquet` filenames, leaving existing `sample_match.parquet` fixtures untouched for non-orientation contract tests. License posture identical to existing precedent (Bassek et al. CC-BY 4.0 for IDSSE; Metrica Sample Game 1 CC-BY-NC-4.0 — same license as existing Sample Game 2 fixture; both excluded from the published wheel).
5. **TF-22 detector hardening (in scope)**: `detect_input_convention` no longer false-positives `ABSOLUTE_FRAME_HOME_RIGHT` on sparse-shot per-period-absolute matches. Validator re-enabled at sportec / metrica / pff converter call sites with `declared=PER_PERIOD_ABSOLUTE`.
6. **Atomic-SPADL pathway smoke test**: Sportec → SPADL → atomic-SPADL composition asserts canonical-LTR invariant survives atomic decomposition. (No converter changes — atomic has no native sportec/metrica.)
7. **ADR-006 erratum** correcting the "Per-converter input convention assignment" table for sportec + metrica + adding a "Lessons learned" section pointing to the missing real-fixture invariant gap.
8. **NOTICE attribution** for the new IDSSE fixture (Bassek et al. CC-BY 4.0) under a new "Test Data Sources" section.
9. **CHANGELOG 3.0.1** with code-snippet migration showing the bool-pair AND the mapping-escape-hatch paths for both providers.
10. **`/final-review` mandatory** before the single PR commit (per project policy).

## 3. Non-goals

- **No native Sportec-XML or Metrica-CSV loader inside silly-kicks.** Existing architectural intent (per `spadl/sportec.py` docstring: "Consumers with raw DFL XML files should use silly_kicks.spadl.kloppy after kloppy.sportec.load_event(...)") is unchanged. Native converters take pre-DataFrame'd events; raw-format ingestion goes through kloppy gateway. New TF-23 logged for follow-up if/when consumer demand surfaces.
- **No change to kloppy-gateway converter (`spadl/kloppy.py`)**. Kloppy normalizes via `Orientation.HOME_AWAY` upstream → ABSOLUTE_FRAME_HOME_RIGHT declaration is correct.
- **No change to Opta converter (`spadl/opta.py`)**. ADR-006 § 2 documents the pre-normalized loader contract; consumer responsibility, no silly-kicks change.
- **No change to PFF converter (`spadl/pff.py`)** beyond re-enabling the (now-hardened) validator. PFF already correctly uses PER_PERIOD_ABSOLUTE.
- **No new ADR.** The fix is an erratum to ADR-006's table — same decision domain, refined contract.
- **No `_finalize_output` per-period orientation assertion.** Per-period orientation requires shots in both periods; insufficient-shot inputs (e.g., a 5-action snippet for unit tests) would either crash or false-pass. The invariant lives at the test layer where fixture density is controlled.
- **No empirical-fallback ergonomic shortcut.** When neither `home_team_start_left` nor `home_attacks_right_per_period` is supplied, the converter raises with a migration message — silly-kicks does not auto-derive per-period direction from event coordinates. Consistent with ADR-006's "single source of truth, no silent self-healing" principle.
- **No change to tracking adapters.** Tracking-side Sportec adapter (`silly_kicks/tracking/sportec.py`) already correctly takes `home_team_start_left` per ADR-006 § 5 (PR-S22). silly-kicks does not ship a native Metrica tracking adapter (kloppy gateway only).

## 4. Architecture

### 4.1 API contract (events-side converters)

Both `silly_kicks/spadl/sportec.py:convert_to_actions` and `silly_kicks/spadl/metrica.py:convert_to_actions` gain three new keyword-only kwargs (positional + existing kwargs unchanged):

```python
def convert_to_actions(
    events: pd.DataFrame,
    home_team_id: str,
    *,
    home_team_start_left: bool | None = None,                          # NEW (3.0.1)
    home_team_start_left_extratime: bool | None = None,                # NEW (3.0.1)
    home_attacks_right_per_period: Mapping[int, bool] | None = None,   # NEW (3.0.1)
    preserve_native: list[str] | None = None,
    goalkeeper_ids: set[str] | None = None,
) -> tuple[pd.DataFrame, ConversionReport]:
```

Resolution policy (mutually exclusive paths):

| `home_team_start_left` | `home_team_start_left_extratime` | `home_attacks_right_per_period` | Behavior |
|---|---|---|---|
| `bool` | `bool` or `None` | `None` | **Preferred path.** Derive flips via `_direction.home_attacks_right_per_period(home_team_start_left, home_team_start_left_extratime)`. ET kwarg is required only when period_id 3/4 is present. |
| `None` | `None` | `Mapping[int, bool]` | **Escape hatch.** Use the mapping directly. For arbitrary-period inputs (futsal halves, friendlies, pre-derived). |
| `bool` or any non-None | any | `Mapping[int, bool]` | `ValueError("pass home_team_start_left[+_extratime] OR home_attacks_right_per_period, not both")`. |
| `None` | `bool` | `None` | `ValueError("home_team_start_left_extratime supplied without home_team_start_left")`. ET flag is meaningful only as a refinement of the regular-time flag. |
| `None` | `None` | `None` | `ValueError` with migration message: explains 3.0.1 contract, points at DFL `MatchInformation.xml` (Sportec) / `metadata.xml` (Metrica), references ADR-006 erratum + CHANGELOG 3.0.1, and shows the bool-pair AND mapping-escape-hatch snippets. |

ET (`home_team_start_left_extratime`) consultation: only when `home_team_start_left is not None` AND the events DataFrame contains `period_id ∈ {3, 4}`. When ET periods are present and the kwarg is None, raise with a clear "ET present, pass home_team_start_left_extratime explicitly" message (matches PFF's existing contract at `spadl/pff.py:304-309`).

### 4.3 Detector hardening (TF-22)

`silly_kicks/spadl/orientation.py:detect_input_convention` currently misclassifies the IDSSE-shape fixture (sparse shots, each team reliable in only one period) as ABSOLUTE_FRAME_HOME_RIGHT. Root cause: at `orientation.py:417-423`, when each team has exactly one reliable (team, period) group, `per_team_sides == 1` is trivially true (one period of data has one side), and the team-means-on-opposite-sides check passes — yielding a false-positive absolute-frame verdict from genuinely per-period-absolute data.

Fix:
1. Track which periods each team has reliable data in.
2. After the `(per_team_sides == 1).all()` branch passes, additionally require that **at least one team has reliable data in ≥2 distinct periods**. If neither team does, the data is per-team-period-asymmetric — cannot distinguish ABSOLUTE_FRAME_HOME_RIGHT from PER_PERIOD_ABSOLUTE on shot-only signal — return `convention=None, confidence="low"` instead.
3. Symmetric guard on the `per_team_sides == 2` branch (already implicitly correct: requires each team to span ≥2 periods, but make the requirement explicit in code + reflect in diagnostics).

After this fix:
- Lakehouse IDSSE shape (each team reliable in 1 period) → `convention=None`, validator silently defers to declared (PER_PERIOD_ABSOLUTE). No false-positive warning.
- Lakehouse Metrica shape (home reliable in 2 periods) → `convention=PER_PERIOD_ABSOLUTE`, validator agrees with declared, no warning.
- Genuine ABSOLUTE_FRAME_HOME_RIGHT data (kloppy gateway post-Orientation.HOME_AWAY) → unchanged classification.

`validate_input_convention` is then re-enabled at all three per-period-absolute converter call sites (sportec, metrica, pff) declaring PER_PERIOD_ABSOLUTE. No call site change semantically — validator was already silent on `convention=None`.

```python
def assert_canonical_ltr_orientation(actions: pd.DataFrame, *, label: str, pitch_mid: float = 52.5) -> None:
    """Assert SPADL canonical LTR convention: every team attacks high-x in their own frame."""
    shot_types = {"shot", "shot_penalty", "shot_freekick"}
    shots = actions[actions["type_name"].isin(shot_types)]
    if shots.empty:
        pytest.skip(f"{label}: no shots in fixture — invariant cannot be physically exercised. "
                    "Either swap to a denser fixture or document the skip with a follow-up TF-row.")

    # Per-period: every (period, team) shot group's mean start_x > pitch_mid.
    per_period = shots.groupby(["period_id", "team_id"])["start_x"].mean()
    bad_per_period = per_period[per_period <= pitch_mid]
    assert bad_per_period.empty, (
        f"{label}: per-period orientation violated. Groups failing canonical LTR:\n"
        f"{bad_per_period.to_string()}\n"
        f"Expected: each (period, team) shot mean > {pitch_mid}; got values <= {pitch_mid}."
    )

    # Aggregate: every team's overall shot mean start_x > pitch_mid (sanity for short fixtures).
    aggregate = shots.groupby("team_id")["start_x"].mean()
    bad_aggregate = aggregate[aggregate <= pitch_mid]
    assert bad_aggregate.empty, (
        f"{label}: aggregate orientation violated. Teams failing canonical LTR:\n"
        f"{bad_aggregate.to_string()}\n"
        f"Expected: each team's overall shot mean > {pitch_mid}; got values <= {pitch_mid}."
    )
```

Distinct failure messages distinguish the two flavors (per-period bug vs whole-fixture bug).

### 4.7 ADR-006 erratum

Append a new section to `docs/superpowers/adrs/ADR-006-direction-of-play-handling.md`:

```markdown
## Erratum (silly-kicks 3.0.1, 2026-05-02)

Two rows of the per-converter input convention assignment table (§ 2)
were incorrect in 3.0.0:

| Converter | declared (3.0.0 — incorrect) | declared (3.0.1 — corrected) |
|---|---|---|
| spadl/sportec.py | ABSOLUTE_FRAME_HOME_RIGHT | PER_PERIOD_ABSOLUTE |
| spadl/metrica.py | ABSOLUTE_FRAME_HOME_RIGHT | PER_PERIOD_ABSOLUTE |

Native Sportec and Metrica bronze events ship per-period-absolute
(teams switch ends after halftime). The kloppy-gateway path
(spadl/kloppy.py) remains ABSOLUTE_FRAME_HOME_RIGHT — kloppy normalizes
upstream via Orientation.HOME_AWAY.

The 3.0.1 fix mirrors the existing PFF events-side and tracking-side
Sportec API exactly: callers pass `home_team_start_left: bool` (or the
escape-hatch `home_attacks_right_per_period` mapping), the converter
derives flips and dispatches to to_spadl_ltr with PER_PERIOD_ABSOLUTE.
See CHANGELOG 3.0.1 for the migration snippet.

The detector heuristic (detect_input_convention) was simultaneously
hardened: when no team has reliable shots in ≥2 distinct periods, the
detector returns convention=None, confidence="low" rather than
false-positiving ABSOLUTE_FRAME_HOME_RIGHT on sparse-shot
per-period-absolute matches. Validator re-enabled at sportec, metrica,
and pff converter call sites with declared=PER_PERIOD_ABSOLUTE.

## Lessons learned (silly-kicks 3.0.1)

PR-S22 reported "1707 passed, 12 skipped, 0 failed" with strict-mode
invariants enabled, yet the Sportec + Metrica per-period bug shipped.
Root cause: tests/datasets/idsse/sample_match.parquet (2 shots total)
and tests/datasets/metrica/sample_match.parquet (period 1 only) had
insufficient per-period shot density to physically exercise the
per-period orientation invariant. The invariant test layer was
present but blind on these two providers.

Lesson: a physical-invariant test is only load-bearing if the fixture
density allows the invariant to actually be evaluated. Future
provider-coverage PRs should verify the invariant suite's fixture
density on a per-provider basis. PR-S23 adds explicit per-period
fixtures (tests/datasets/{idsse,metrica}/per_period_match.parquet)
with documented shot density requirements (≥5 shots per (team,
period) group is the validator's medium-confidence threshold; the
new fixtures meet this).
```

### 4.8 CHANGELOG 3.0.1 entry

```markdown
## 3.0.1 — 2026-05-02 — Direction-of-play correctness fix (Sportec + Metrica)

### Breaking-correctness fix

`silly_kicks.spadl.sportec.convert_to_actions` and
`silly_kicks.spadl.metrica.convert_to_actions` now correctly handle
per-period-absolute bronze events (teams switching ends after
halftime). silly-kicks 3.0.0 declared these converters as
ABSOLUTE_FRAME_HOME_RIGHT, producing wrong-end SPADL output for half
of every match. ADR-006 erratum documents the corrected per-converter
declaration table.

Callers must now pass per-period direction info via one of two paths
(otherwise ValueError with migration guidance):

# Path A — bool pair (preferred; matches PFF events + tracking-Sportec API)
actions, report = sportec.convert_to_actions(
    events,
    home_team_id="DFL-CLU-XXX",
    home_team_start_left=True,                     # from DFL MatchInformation.xml
    home_team_start_left_extratime=False,          # only when ET periods present
)

# Path B — explicit mapping (escape hatch for arbitrary periods)
actions, report = metrica.convert_to_actions(
    events,
    home_team_id="HomeTeam",
    home_attacks_right_per_period={1: True, 2: False},
)

Trained VAEP / HybridVAEP / xT models on Sportec or Metrica data from
silly-kicks 3.0.0 must be re-trained on 3.0.1 output.

### Test infrastructure
```
